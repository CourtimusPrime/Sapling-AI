## Codebase Patterns
- `@tanstack/store` v0.8.1: `subscribe` returns `() => void` (cleanup function directly); listener receives `{ prevVal, currentVal }` object — use `({ currentVal }) => ...`; `setState` takes updater `(prev: T) => T` (unchanged)
- Untyped npm packages in Deno: use `// @deno-types="npm:@types/package@version"` pragma above the import; BUT the @types package must also be in deno.json imports so `deno install` tracks it and doesn't remove it later
- App-wide client state (activeNodeId, activeChatId, viewport) lives in `stores/chat.ts` (`appStore`); islands subscribe via `appStore.subscribe(({ currentVal }) => ...)` returning a cleanup `() => void`
- `@tanstack/store` v0.8.1 subscribe listener receives `{ prevVal, currentVal }` (NOT raw state); use `const unsub = appStore.subscribe(({ currentVal }) => ...); return unsub;` — the returned value is already a cleanup function
- TanStack Query in islands: each island creates its own `QueryClient` (module-level const) wrapped in `QueryClientProvider`; islands cannot share QueryClient context since they hydrate independently
- `@tanstack/preact-query` v5: `useQuery`, `useMutation`, `QueryClient`, `QueryClientProvider` all exported from `@tanstack/preact-query`; `useMutation` uses `isPending` not `isLoading`; `onSuccess: (data, variables, context) => void`
- Biome `useSemanticElements`: `<div role="button">` is a lint error — use `<button type="button">` instead; action buttons can be siblings (absolutely positioned) rather than nested, to avoid invalid HTML button-in-button nesting
- Cascade delete for chats: manually delete node_metadata (inArray nodeIds) → node (chatId) → chat — no onDelete cascade in schema
- Chat/node ownership check: `and(eq(table.id, id), eq(table.userId, userId))` to verify the resource belongs to the requesting user


- AES-GCM encryption: `lib/crypto.ts` exports `encrypt(plaintext)` / `decrypt(encoded)` — key derived via HKDF from `ENCRYPTION_KEY` env var; ciphertext stored as base64url(iv || ciphertext)
