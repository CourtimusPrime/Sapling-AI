## Codebase Patterns
- Custom headers on SSE responses: call `c.header(name, value)` before `streamSSE(c, ...)` ŌĆö they appear in `res.headers` on the SSE client side (same-origin)
- Cross-island node refresh: `appStore.nodeRefreshTrigger` counter incremented by ChatPanel after stream; Mindmap subscription detects change and increments local `refreshKey`; `refreshKey` in fetch `useEffect` deps triggers re-fetch. `biome-ignore` must go on the line ABOVE `useEffect(`, not inside the callback.
- SSE stream parsing via fetch: `res.body.getReader()` ŌåÆ loop `reader.read()` ŌåÆ `TextDecoder.decode(value, { stream: true })` ŌåÆ split on `\n`, keep tail (`lines.pop()`) for incomplete lines ŌåÆ filter `line.startsWith('data: ')` ŌåÆ slice 6 chars to get payload.
- Throttled `useState` setter in component: `useRef(throttle(setter, { wait: 16 })).current` ŌĆö ref initializer runs once, captures stable setter; use for streaming content batching at ~60fps.
- AI SDK v4: `streamText(...)` NOT awaited ŌĆö synchronous return; `result.textStream` is `AsyncIterable<string>`; `result.usage` is `Promise<{ promptTokens, completionTokens, totalTokens }>`; providers: `createOpenAI`, `createAnthropic` from `@ai-sdk/openai`, `@ai-sdk/anthropic`; OpenRouter uses `createOpenAI({ apiKey, baseURL: 'https://openrouter.ai/api/v1' })`
- `streamSSE` from `hono/streaming` for SSE responses (`Content-Type: text/event-stream`); the async callback is fully awaited ŌĆö DB writes after the stream loop complete before response closes
- `@tanstack/store` v0.8.1: `subscribe` returns `() => void` (cleanup function directly); listener receives `{ prevVal, currentVal }` object ŌĆö use `({ currentVal }) => ...`; `setState` takes updater `(prev: T) => T` (unchanged)
- Untyped npm packages in Deno: use `// @deno-types="npm:@types/package@version"` pragma above the import; BUT the @types package must also be in deno.json imports so `deno install` tracks it and doesn't remove it later
- App-wide client state (activeNodeId, activeChatId, viewport) lives in `stores/chat.ts` (`appStore`); islands subscribe via `appStore.subscribe(({ currentVal }) => ...)` returning a cleanup `() => void`
- `@tanstack/store` v0.8.1 subscribe listener receives `{ prevVal, currentVal }` (NOT raw state); use `const unsub = appStore.subscribe(({ currentVal }) => ...); return unsub;` ŌĆö the returned value is already a cleanup function
- TanStack Query in islands: each island creates its own `QueryClient` (module-level const) wrapped in `QueryClientProvider`; islands cannot share QueryClient context since they hydrate independently
- `@tanstack/preact-query` v5: `useQuery`, `useMutation`, `QueryClient`, `QueryClientProvider` all exported from `@tanstack/preact-query`; `useMutation` uses `isPending` not `isLoading`; `onSuccess: (data, variables, context) => void`
