## Codebase Patterns
- requireAuth middleware is in lib/auth.ts; exports AuthEnv type ({ Variables: { user: { id, email } } }) used to type Hono instances and sub-apps
- Protected routes use a sub-app: `const protectedRoutes = new Hono<AuthEnv>(); protectedRoutes.use("/*", requireAuth);` mounted via `api.route("/", protectedRoutes)` in api/app.ts
- hono/factory's `createMiddleware<Env>()` is how middleware with typed context variables is created in Hono v4
- `deleteCookie(c, name, { path: "/" })` from hono/cookie clears a cookie in Hono
- Access typed context variables with `c.var.user` (shorthand for `c.get("user")`) once the Hono instance is typed with the correct Env
- This project uses Fresh 2 (not Fresh 1). Fresh 2 uses vite.config.ts instead of fresh.config.ts, and does not include a dev.ts — the dev server runs via `deno task dev` which calls vite directly.
- Fresh 2 app entry point is main.ts (creates App instance, adds middleware, calls app.fsRoutes())
- utils.ts uses createDefine<State>() to create typed helpers (define.page, define.handlers, define.middleware)
- deno task dev = vite dev server; deno task build = vite build; deno task start = deno serve _fresh/server.js
- Drizzle schema uses `drizzle-orm/sqlite-core` (sqliteTable, text, integer, real, index, primaryKey); all tables in db/schema.ts
- drizzle.config.ts uses `process.env` (runs via drizzle-kit in Node.js) and is excluded from Deno typecheck via deno.json exclude array
- db:generate = `node ./node_modules/.bin/drizzle-kit generate`; db:migrate = `deno run -A db/migrate.ts` (uses drizzle-orm/libsql/migrator)
- Biome auto-fix: run `./node_modules/.bin/biome check --write .` to auto-fix formatting issues from generated files (JSON snapshots, new TS files)
- Pre-commit check: `deno check **/*.ts **/*.tsx` from versions/deno-vs/ for typecheck
- Hono API app lives in api/app.ts, uses `.basePath("/api")`, routes defined without /api prefix (e.g. api.get("/health", ...))
- Fresh 2 define.handlers() accepts either a method-keyed object OR a plain function (HandlerFn) that handles all HTTP methods
- Catch-all Fresh route for Hono: routes/api/[...path].ts with `define.handlers((ctx) => api.fetch(ctx.req))`
- `nodeModulesDir: "manual"` requires running `deno install` after adding npm dependencies to deno.json
- Deno selects `@libsql/client` web implementation which does NOT support `file:` URLs — use `@libsql/client/node` (add as separate import map entry `"@libsql/client/node": "npm:@libsql/client@^0.14.0/node"`) which uses native bindings and supports `file:./sapling.db` for local dev
- Drizzle client is in `db/client.ts`, exports `db` — uses `drizzle-orm/libsql` driver with `@libsql/client/node`
- UnoCSS: add `"unocss": "npm:unocss@^0.65.3"` and `"unocss/vite": "npm:unocss@^0.65.3/vite"` to deno.json imports; configure in vite.config.ts (not fresh.config.ts); import `virtual:uno.css` in client.ts
- Biome: add `"@biomejs/biome": "npm:@biomejs/biome@^1.9.4"` to deno.json; run `deno install --allow-scripts=npm:@biomejs/biome` (regular deno install ignores the postinstall script, no binary is created); `deno task lint` = `./node_modules/.bin/biome check .`
- Auth: `jose` (npm:jose@^5) for JWT signing/verification; `zod` (npm:zod@^3) for request validation; password hashing via PBKDF2 in lib/hash.ts (Web Crypto API); auth routes in api/routes/auth.ts, mounted via `api.route("/auth", authRouter)` in api/app.ts
- Drizzle select single row: `const [row] = await db.select(...).from(table).where(...).limit(1);` (async, returns array)
- JWT cookie: `setCookie(c, "auth", token, { httpOnly: true, sameSite: "Lax", path: "/", maxAge: ... })` from `hono/cookie`
